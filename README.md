# remi-cat

An AI agent bot that connects to Feishu (Lark) and runs in a daemon/agent architecture with an admin UI.

## Quick Start

Run the interactive setup script — it handles prerequisites, builds, configuration, and secrets:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/another-s347/remi-cat/main/setup.sh)
```

The setup script will:

1. Check and install prerequisites (Rust, protobuf-compiler, Docker, Docker Compose)
2. Ask which deployment mode you want:
   - **Standalone** — single binary, no Docker required
   - **Daemon + Agent** — recommended production setup with Docker
3. Configure ports, data directory, and Docker bind mounts
4. Prompt for API secrets (hidden input — never stored in plain text)
5. Write `.env` with all configuration
6. In daemon mode: import secrets into the AES-256-GCM encrypted secret store and strip them from `.env`

## Architecture

| Binary | Description |
|---|---|
| `remi-cat` | Standalone bot process |
| `remi-cat-agent` | Agent process (daemon mode) |
| `remi-daemon` | Daemon that manages the agent container |
| `remi-admin` | Admin UI server |

## Configuration Files

The following files are **not committed to the repository** and are generated by `setup.sh`:

- `.env` — environment variables and configuration
- `owner.json` — bot owner identity
- `mgmt_identity.json` — management channel identity

## Building Manually

```bash
cargo build --release
```

Binaries are output to `target/release/`.
